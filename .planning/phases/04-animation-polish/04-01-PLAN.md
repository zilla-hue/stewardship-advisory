    **Step 2 — Wrap Outlet in AnimatePresence in RootLayout.tsx (ANIM-01):**

    Update `src/layouts/RootLayout.tsx`:
    1. Add imports: `import { AnimatePresence, motion } from "motion/react";` and `import { pageTransition } from "../lib/animations";`
    2. Replace the `<main><Outlet /></main>` block with:
       ```tsx
       <AnimatePresence mode="wait">
         <motion.main
           key={location.pathname}
           {...pageTransition}
         >
           <Outlet />
         </motion.main>
       </AnimatePresence>
       ```
    3. Keep the existing `useEffect` for `window.scrollTo(0, 0)` — the 0.25s exit duration is short enough that the scroll snap is not noticeable.
    4. Keep Navbar and Footer OUTSIDE AnimatePresence — they are persistent layout elements.

    **Step 3 — Fix first-section animation triggers on 3 components (ANIM-02):**

    These components are the first (or only) section rendered on their respective pages. Their outer `motion.div` must use `animate` instead of `whileInView` so the entrance animation fires on mount regardless of page height.

    **About.tsx** — Change the outer `motion.div` (line 7-10):
    - Remove: `whileInView="visible"` and `viewport={{ once: true, margin: "-100px" }}`
    - Add: `animate="visible"`
    - Keep: `initial="hidden"` and `variants={staggerContainer}`
    - Do NOT change any child `motion.div variants={fadeUp}` elements — they animate via stagger from parent.

    **Services.tsx** — Change BOTH `motion.div` containers:
    - First container (line 47-52, the header section): Remove `whileInView="visible"` and `viewport={{ once: true, margin: "-100px" }}`, add `animate="visible"`. Keep `initial="hidden"` and `variants={staggerContainer}`.
    - Second container (line 77-83, the accordion section): Same change — remove `whileInView` and `viewport`, add `animate="visible"`. This section is immediately below the header on ServicesPage so it is visible on mount at most viewport sizes.

    **Values.tsx** — Change BOTH `motion.div` containers:
    - First container (line 48-53, heading section): Remove `whileInView="visible"` and `viewport={{ once: true, margin: "-100px" }}`, add `animate="visible"`.
    - Second container (line 69-74, grid section): Same change. Values is the only component on ValuesPage, so the grid is likely in viewport on mount.

    **Do NOT change** these components (they are correct as-is):
    - Hero.tsx — already uses `animate="visible"` (correct)
    - ContactPage.tsx — already uses `animate="visible"` (correct)
    - VisionMission.tsx — second section on AboutPage, `whileInView` is appropriate
    - Framework.tsx — second section on ServicesPage, `whileInView` is appropriate
    - CTA.tsx — last section on HomePage, `whileInView` is appropriate

    **Important:** These components are also used on the HomePage where they are NOT the first section. Changing `whileInView` to `animate` means they will animate on mount even when scrolled off-screen on HomePage. This is acceptable because: (a) the animations are opacity + y-transform which settle quickly, (b) the components use `staggerContainer` which means children animate sequentially — by the time the user scrolls down, animations have completed. The alternative (passing a prop to conditionally use whileInView vs animate) adds complexity for marginal benefit. Keep it simple.
