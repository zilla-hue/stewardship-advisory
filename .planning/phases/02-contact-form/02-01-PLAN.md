The component stays in ContactPage.tsx (not extracted to a separate file — Phase 1 established that page components own their content directly).

    **Imports:**
    ```typescript
    import { useState } from "react";
    import { useForm, ValidationError } from "@formspree/react";
    import { motion } from "motion/react";
    import { Loader2 } from "lucide-react";
    import { fadeUp, staggerContainer } from "../lib/animations";
    ```

    **Constants (define above the component):**
    ```typescript
    const INQUIRY_TOPICS = [
      "Institutional Communications Strategy",
      "Reputation Advisory",
      "Stakeholder Engagement",
      "Crisis Communications",
      "I-TRUST Framework",
      "General Inquiry",
    ];
    ```

    **Component structure (const ContactPage arrow function):**

    1. **useForm hook:** `const [state, handleSubmit] = useForm("xpwrggkj");` -- Use the placeholder ID `"xpwrggkj"` which will be replaced with the real Formspree endpoint ID during the human-verify step. A non-empty string is required to prevent the "You must provide a form key" crash (per RESEARCH.md Pitfall 1).

    2. **Local state for controlled inputs:**
       - `const [name, setName] = useState("")`
       - `const [email, setEmail] = useState("")`
       - `const [organization, setOrganization] = useState("")`
       - `const [topic, setTopic] = useState("")`
       - `const [message, setMessage] = useState("")`
       - `const [clientErrors, setClientErrors] = useState<Record<string, string>>({})`

    3. **validateAndSubmit function** (per CONT-02):
       - Call `e.preventDefault()`
       - Build `errs` object: check `name.trim()` empty -> `"Name is required"`, check `email.trim()` empty -> `"Email is required"`, else check email against regex `/^[^\s@]+@[^\s@]+\.[^\s@]+$/` -> `"Please enter a valid email address"`, check `message.trim()` empty -> `"Message is required"`
       - If any errors: `setClientErrors(errs); return;`
       - If no errors: `setClientErrors({}); handleSubmit(e);`

    4. **Success state rendering** (per CONT-04 — replaces entire form):
       - If `state.succeeded` is true, return a `<section>` with the same wrapper classes as the form section (`bg-white text-[#0A1628] py-24 md:py-32 min-h-screen`)
       - Inside: `motion.div` with `initial="hidden" animate="visible" variants={staggerContainer}` and container class `max-w-7xl mx-auto px-6 md:px-12`
       - `motion.h2` with `variants={fadeUp}` class `text-4xl font-serif mb-6` text: "Message received"
       - `motion.p` with `variants={fadeUp}` class `text-[#0A1628]/70 font-light` text: "Thank you for reaching out. We will be in touch shortly."
       - Use `animate="visible"` NOT `whileInView` (mount-triggered, per Phase 1 decision for short Contact page)

    5. **Form rendering** (the default/main return):
       - Outer `<section className="bg-white text-[#0A1628] py-24 md:py-32 min-h-screen">`
       - `motion.div` container with `initial="hidden" animate="visible" variants={staggerContainer}` and `className="max-w-7xl mx-auto px-6 md:px-12"`
       - Preserve existing heading block from the shell:
         - `motion.div` with `variants={fadeUp}` mb-4: `<span>` "Contact" label (text-xs font-bold tracking-widest uppercase text-[#0A1628]/40)
         - `motion.h1` with `variants={fadeUp}`: "Get in Touch" (text-4xl md:text-5xl font-serif mb-8)
         - `motion.p` with `variants={fadeUp}`: "We welcome inquiries..." paragraph (text-lg text-[#0A1628]/70 font-light leading-relaxed mb-12 max-w-2xl)
       - `motion.form` with `variants={fadeUp}` `onSubmit={validateAndSubmit}` `noValidate` `className="max-w-2xl space-y-6"`
       - Five field groups inside the form, each in a `<div>`:

         **Field 1 - Full Name (required):**
         - `<label htmlFor="name">` text "Full Name" with `<span className="text-red-500">*</span>`
         - `<input id="name" name="name" type="text" value={name} onChange={(e) => setName(e.target.value)}>`
         - Input className: `w-full border border-[#0A1628]/20 rounded px-4 py-3 text-[#0A1628] bg-white focus:outline-none focus:border-[#0A1628]`
         - Conditional error: `{clientErrors.name && <p className="text-red-500 text-sm mt-1">{clientErrors.name}</p>}`

         **Field 2 - Email Address (required):**
         - `<label htmlFor="email">` text "Email Address" with red asterisk
         - `<input id="email" name="email" type="email" value={email} onChange={(e) => setEmail(e.target.value)}>`
         - Same input className as name field
         - Client error: `{clientErrors.email && <p>...</p>}`
         - Server error: `<ValidationError field="email" errors={state.errors} className="text-red-500 text-sm mt-1" />`

         **Field 3 - Organisation (optional, no asterisk):**
         - `<label htmlFor="organization">` text "Organisation" (no asterisk)
         - `<input id="organization" name="organization" type="text" value={organization} onChange>`
         - Same input className

         **Field 4 - Inquiry Topic (optional select dropdown):**
         - `<label htmlFor="topic">` text "Inquiry Topic" (no asterisk)
         - `<select id="topic" name="topic" value={topic} onChange>`
         - Same className as inputs
         - Default `<option value="">Select a topic</option>`
         - Map INQUIRY_TOPICS to `<option key={t} value={t}>{t}</option>`

         **Field 5 - Message (required):**
         - `<label htmlFor="message">` text "Message" with red asterisk
         - `<textarea id="message" name="message" rows={5} value={message} onChange>`
         - className adds `resize-none` to the standard input className
         - Client error for message

       - **Form-level Formspree error** (after all fields, before button):
         `<ValidationError errors={state.errors} className="text-red-500 text-sm" />`

       - **Submit button:**
         `<button type="submit" disabled={state.submitting} className="inline-flex items-center gap-2 px-10 py-4 rounded-full bg-[#0A1628] text-white font-medium hover:bg-[#0A1628]/90 transition-colors disabled:opacity-60">`
         - When `state.submitting`: render `<Loader2 className="w-4 h-4 animate-spin" />` + text "Sending..."
         - When not submitting: render text "Send Message"

    6. **Label className** (consistent across all 5 fields): `block text-sm font-medium mb-1`

    7. **Default export:** `export default ContactPage;`

    **Important implementation notes:**
    - All input/textarea/select elements MUST have explicit `text-[#0A1628] bg-white` to prevent white-on-white text (per RESEARCH.md Pitfall 5, body has color: #ffffff)
    - Use `animate="visible"` everywhere, NOT `whileInView` (per Phase 1 decision — Contact page is short, scroll trigger never fires)
    - The `noValidate` attribute on the form element disables browser-native validation bubbles so our custom inline errors show instead
    - The `name` attribute on each input is REQUIRED for Formspree to receive the field data in the submission email
